Grunt 初步学习

发表于 2017-09-14 | 分类于 Grunt

Grunt 自动化构建

Grunt 和所有 Grunt 插件都是基于 nodejs 来运行的

安装 CLI

直接使用 npm 包管理器来安装 grunt-cli 工具

当然这里是在全局安装了 grunt-cli 工具，并配置了 grunt 命令对应的 path 路径，那么就可以在任何目录下运行 grunt 了。

然后呢，当我们在终端敲命令：

却提示 Fatal error: Unalble to find local grunt.

原因是：安装了 grunt-cli 并不等于安装了 Grunt —— JavaScript Task Runner。

grunt-cli 的任务只是：调用与 Gruntfile 在同一目录中的 grunt，因此在同一台机器上可以同时安装多个不同版本的 grunt。

在项目中安装 Grunt 及要使用到的 Grunt 插件

在需要自动构建的项目中安装 grunt

grunt 是要配合着其 grunt 插件使用的，根据任务的需求，需要用到的有：

  • 合并文件: grunt-contrib-concat
  • 语法检查: grunt-eslint
  • Scss编译: grunt-contrib-sass
  • 压缩文件: grunt-contrib-uglify
  • 监听文件变动: grunt-contrib-watch

配置 Gruntfile 文件

在项目根目录下创建 Gruntfile.js 或者 Gruntfile.coffee 文件，即 package.json 同目录

一个 Gruntfile 文件由以下几部分组成:

  • wrapper 函数
  • 项目和任务配置
  • 加载 Grunt 插件和任务
  • 自定义的任务

运行任务

格式为 grunt <任务名>，任务名为空的话，就是默认执行 default 任务。

这样就把 src 目录下的 styles 目录下的 .scss 后缀文件编译到 build 目录下生成 css 文件

Tips

  • grunt-contrib-sass 使用必须安装 ruby 和 sass。sass 必须是 ruby 版的。
