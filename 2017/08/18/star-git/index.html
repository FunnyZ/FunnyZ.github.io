学会使用 Git 及 Git 常用命令

发表于 2017-08-18 | 分类于 Git

Git 是免费的、开源的分布式版本管理系统。它涉及到多人合作、分支、备份、恢复、打标签等诸多有趣的命令，正是这些使 Git 带给了我无限的乐趣。接下来，我就来说说我自己的拙见。

快速开始吧

安装 Git

安装的话非常的方便，直接到官网上下载就可以了，Windows 的话是包含 Git Bash 和 Git GUI 两种使用方式的。Git Bash 就是敲命令，Git GUI 就是图像化界面的（详情操作我是没弄过，你们可以玩玩）。Mac 的话就是终端命令，当然也可以下载客户端。

当然任何命令我们都可以先用 –help 来看下帮助

初始化，创建新仓库

这样一个新的仓库就创建出来了，目录下会多出个东西出来

  • 一个隐藏文件夹：.git（看到它就是创建成功了，里面的 config 文件就是 git 的基础配置）

有了仓库，你可以检出已有的仓库，或者重新着手

  • 检出本地仓库的克隆版

  • 检出远程仓库

简单说下工作流

首先你是在你的工作目录，它持有实际的文件，也就是文件夹下进行文件及其内容的增删改查操作；接着是一个暂存区，就是 git 给你保存工作现场的一个缓存区域，临时保存你的改动；最后才是提交到你的当前记录上。

  • 查看状态（即查看是否有改动）

  • 有改动，把改动添加到暂存区（分添加、删除）

  • 确认提交，git 会产生一条提交记录

一般都是跟远程仓库绑定使用的

  • 绑定远程仓库

  • 本地仓库推送到远程仓库

  • 若是远程仓库上已有本地仓库不一致的文件，则要先拉取该分支内容

  • 拉取分支后，合并分支，可能会自动合并，但若存在冲突，便要手动解决

  • 所以呢，最好先查看下有什么不同

多人合作，最好不要在同一分支上同时操作

  • 分支操作

推荐打标签

  • 打标签，可以更清晰地让我们知道是哪一版本

More 命令

  • 历史纪录

  • 回退操作

  • 查看远程仓库信息

  • 查看配置

  • 配置设置

    • 配置别名

    • 设置用户名

    • 设置电子邮箱
